# Agenda 2. 山口の事業仮説 — 「自分はこうする」

## 2-1. 判断の核心 — 「需要があること」と「自分がそこに食い込めること」は別の話

3C分析だけを見れば、BPOが合理的に見える。しかし「Company（自社）」の解像度をもう一段上げると、評価が変わる。

**それが「山口個人のアセット」という変数。**

| アセット | BPOでの活用 | 観光テックでの活用 |
|---------|-----------|-----------------|
| ベトナム上流階級との既存コネクション | **活用不可**（マレーシア日系企業向けの事業） | **直接活用**（デマンド側の初期顧客に直結） |
| ベトナム語コミュニケーション能力 | 間接的（BPO人材採用に一部活用） | **直接活用**（サプライ・デマンド両面） |
| マレーシア日系企業とのコネクション | **ゼロ。完全なゼロスタート** | 不要 |

**率直に言う。** BPO事業は市場も空白もある。しかし「コネクションがベトナムにしかない山口」がマレーシアでゼロから顧客を開拓する事業計画は、構造的に弱い。

- BPOを選んだ場合の最初のアクション：JACTIM加入 → 月例会に参加 → 名刺交換 → 3ヶ月かけて関係構築 → やっと商談化。
- 観光テックを選んだ場合の最初のアクション：自分の既存ネットワークにテストツアーを提案。**Day 1から動ける。**

> 3C分析の「Company」で Harry& のアセット（AI技術・採用計画）を見ればBPOが有利。
> しかし、**事業を実際に立ち上げて最初の売上を作るのは山口であり、山口の武器はベトナム側にしかない。**

## 2-2. 山口の推奨：観光テック事業に全集中

### なぜ「プラットフォーム」なのか

佐々木氏が以前「テック系ソフトウェアでやるならシナジーがある」と言った意味を、自分なりに解釈した。

| やり方 | Harry&のシナジー | 結論 |
|--------|----------------|------|
| ガイド派遣業（人を雇って案内する） | **ゼロ。** IT基盤不要。Harry&は無関係 | ✗ これではHarry&と組む意味がない |
| **テックプラットフォーム（予約管理・CRM・決済・AI行程生成）** | **全面活用。** AWS基盤、システム連携、事業戦略設計のすべてが必要 | **◎ Harry&の自社プロダクトとして位置づけられる** |

**つまり、観光テック事業は「Harry&の自社プロダクト開発」。** クライアントワークではなく、受託以外の収益柱への挑戦。

→ **この位置づけで佐々木氏が合意できるかが、今日の最大の論点。**

---

### 事業の全体像

| 要素 | 内容 |
|------|------|
| **サプライ側（ガイド）** | 在日ベトナム人（66万人）から高品質ガイドを募集・審査・登録 |

## 2-3. BPO事業 — 選ばない理由の補足

BPOを否定しているわけではない。3C上の空白は存在し、Harry&のアセットとの適合度も高い。

**しかし「今の山口が」やるべき事業ではないと判断した。**

| 論点 | BPOの現実 |
|------|----------|
| **顧客アクセス** | マレーシア日系企業1,600社とのコネクション = **ゼロ。** JACTIM加入→関係構築2-3ヶ月→商談化さらに2-3ヶ月。最速でも初回売上は6ヶ月後 |
| **初動の差** | 観光テックはDay 1から既存ネットワークで動ける。BPOはDay 1にJACTIMの入会申請書を書くところから |
| **リスクの質** | 観光テックのリスクは「テック開発投資の回収」＝佐々木氏と共にコントロール可能。BPOのリスクは「ゼロからの関係構築」＝山口の営業力という不確実性に依存 |

> BPO事業は「Harry&の誰かが」やるべき事業かもしれないが、「コネクションがベトナムにしかない山口が」ゼロから立ち上げる事業としては、構造的に弱い。

## 2-4. 佐々木氏がGo/No-Goを判断するための比較表

佐々木氏の立場で「何があれば判断できるか」を考えて整理した。

| 判断項目 | BPO事業 | 観光テック事業 |
|---------|---------|-------------|
| **初期投資** | ほぼゼロ（営業費のみ）。構築費はクライアント持ち | 開発費（Harry&工数。MVP段階で200-500万円相当） |
| **初回売上時期** | 最速3ヶ月（顧客獲得次第） | 6ヶ月（テスト運用後） |
| **12ヶ月後の月間売上** | 3社×Standard = RM15,000（≒40万円/月） | GMV 144万円 × 30% = 43万円/月 |
| **Harry&へのインパクト** | 初期構築費100-300万円/社＋月額運用の70% | **自社プロダクトとしての開発投資。成功すれば受託以外の収益柱** |
| **最大のリスク** | 山口のMY日系企業コネクション = ゼロ | テック開発投資の回収不確実性 |
| **撤退基準** | 3ヶ月で5社ヒアリング中0社が関心なし → 撤退 | ガイド登録50名未満 or テスト予約3件未満 → 撤退 |
| **佐々木氏の負荷** | 低（提案サポート＋通常案件対応） | 中（開発の意思決定・設計レビュー） |

## 3-1. 最重要質問 — 自社プロダクト投資への意思

**今日の最大の論点。ここが決まれば、残りはすべて実行計画の話。**

> **「Harry&として、受託開発ではなく自社プロダクトとしてプラットフォームを開発することに投資する意思はあるか？」**

| 背景 |
|------|
| 観光テック事業は「ガイド派遣業」ならHarry&不要。「テックプラットフォーム」として構築して初めてHarry&の全スキルが活きる。受託100%のポートフォリオに自社プロダクトを加えるのは、Harry&にとっても大きな意思決定 |

### 回答パターン別の山口のアクション

| 佐々木氏の回答 | 山口のネクストアクション |
|--------------|---------------------|
| **「投資する。自社プロダクトとしてやろう」** | 観光テック事業に全集中。90日ロードマップを即日開始 |
| **「投資は難しい。受託の延長でしか考えていない」** | MVP段階は外部ツール（Googleフォーム＋Zapier等）で代替し、テスト結果で再度投資判断を仰ぐ |
| **「観光テックよりBPOの方がHarry&としてはやりたい」** | BPO事業に切り替え。顧客開拓ゼロスタートの解決策を佐々木氏と共に練る（→ Q4参照） |
